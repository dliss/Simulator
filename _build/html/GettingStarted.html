Getting Started¶

The Simulator package provides a set of python modules for simulating sources in the gamma-ray sky. A basic simulation can be generated by running the Controller module with the default options. More sophisticated simulations can be developed by adapting the modules and options to your specific project.

Within the software, a Source module stores parameters for each simulated source, while a Model module stores the functions that generate each source’s light curve. In both modules, a Standard class is provided for generic sources and subclasses are provided for more specific or sophisticated sources.

The SourceMap module stores multiple Sources when more than one sources are simulated. Meanwhile, the SourceGenerator module provides a framework for runing multiple Sources in a single simulation. The software also comes with a Plotter module to help users analyze their results and a Controller class to call the other modules.

Source¶

The Simulator software comes with the following Source classes:

class Source.Standard(startTime=1000, endTime=1296600, timeRes=10000, upperE=10000.0, lowerE=100.0, energyRes=100, gaLong=0.0, gaLat=0.0, link='modelParameters.txt', useFile='no', skipPrompts='no')¶

Source.Standard represents a generic Source object with the most basic parameters.

class Source.Transient(emissionStart=4000, emissionLength=1209600, startTime=1000, endTime=1296600, timeRes=10000, upperE=10000.0, lowerE=100.0, energyRes=100, gaLong=0.0, gaLat=0.0, link='modelParameters.txt', useFile='no', skipPrompts='no')¶

Source.Transient represents a transient Source object with emission starting at emissionStart and lasting for emissionLength.

class Source.VaryingSpectrum(startTime=1000, endTime=1296600, timeRes=10000, upperE=10000.0, lowerE=100.0, energyRes=100, gaLong=0.0, gaLat=0.0, link='modelParameters.txt', useFile='no', skipPrompts='no')¶

Source.VaryingSpectrum represents a Source object with varying spectrum.

Source.Standard represents a generic Source object, while all other Source classes inherit from the standard Source object to define more specific kinds of sources.

The standard Source object provides the followings parameters for defining a source:

  • startTime – start time of simulation
  • endTime – end time of simulation
  • timeRes – time resolution
  • upperE – upper energy limit
  • lowerE – lower energy limit
  • energyRes – energy resolution
  • gaLong – positional information (Galactic Longitude)
  • gaLat – positional information (Galactic Latitude)
  • link – the file location from which parameters can be written and read
  • useFile – indicates whether spectrum should be read from a file
  • skipPrompts – indicates whether default analytical expressions should be used to define a source’s spectrum and time-dependent flux.

The standard Source object can call the following methods:

Standard.writeP()¶

writes the Source’s parameters to a csv file

Standard.readP()¶

reads the Source’s parameters from a csv file

Standard.store_flux_energy_list(array)¶

stores a light curve inside the Source object

Standard.get_coord()¶

returns the positional information for a source in the regions file format

Standard.prompt_for_expressions()¶

prompts the user for the general expressions that define the light curve of a Source.

Source.Transient provides new emissionStart and emissionLength parameters to indentify the start time of a transient’s emission and its duration. Transient also overrides the prompt_for_expressions method to suggest a set of functions that will simulate the flux and spectrum of a generic, constant spectrum, novae.

Transient.prompt_for_expressions()¶

overrides the Standard method to suggest general expressions that may be convenient in Transient simulations

Source.VaryingSpectrum needs no new parameters but overrides the prompt_for_expressions method so that the user will not be prompted for spectrum parameters. (For this subclass, Model.VaryingSpectrum is responsible for defining the spectrum.)

VaryingSpectrum.prompt_for_expressions()¶

overrides the Standard method to suggest general expressions that may be convenient in VaryingSpectrum simulations and to override the prompts for spectral parameters

Model¶

The Simulator software comes with the following Model classes:

class Model.Standard(source)¶

contains the standard set of methods for generating the light curve of a Standard source object.

class Model.Transient(source)¶

contains the specific methods for generating the light curve of a Transient source object.

class Model.VaryingSpectrum(source)¶

contains the specific methods for generating the light curve of a source with varying spectral shape. – NOTE: user must edit the file names and time intervals in the method called spectral_shape_from_file to control the time-dependent spectral shape of a source’s light curve

Model.Standard represents a generic Model that can generate the light curve of a standard Source, while all other Model classes inherit from the standard Model to define more specific kinds of Models. These models correspond with the Source objects of similar name.

The standard Model object can call the following methods:

Standard.log_this()¶

handles all the logging for the Model

Standard.generate_flux_values()¶

generates flux values for specified time steps, without breaks in emission, using the f_time function.

Standard.generate_spectral_shape()¶

generates constant spectral shape for a light curve, using the f_spec function.

Standard.generate_light_curve_from_flux_and_spectrum()¶

creates the array that stores the finished light curve, after generate_flux_values and generate_spectral_shape are run

Standard.spectral_shape_from_file()¶

support function for generate_spectral_shape, which takes input from a file

Standard.make_spec(expr)¶

creates the analytical expression which defines a source’s spectrum

Standard.make_func(expr)¶

creates the analytical expression which defines a source’s time-dependent flux

Standard.warn_if_errors_present()¶

checks the settings provided by the user for potential errors, logs if errors present

Standard.check_parens(string)¶

support method for warn_if_errors_present

Standard.sample_spectrum_from_file()¶

prints a template for the spectrum file

Model.Transient differs from the standard Model in how it generate the time-dependent flux of a source (it shuts off the emission before and after the transient’s flare). To achieve this effect, Model.Transient overrides the generate_flux_values method of the standard Model.

Transient.generate_flux_values()¶

Model.Transient differs from the standard Model because it generates the spectrum of a source using time intervals and txt files specified in the spectral_shape_from_file method. It also overrides several other methods that are used in Model.Standard to generate constant spectral shape. (NOTE: User must edit the subclass’s spectral_shape_from_file method if they would like to control the time intervals and txt files used in the simulations).

VaryingSpectrum.generate_spectral_shape()¶

overrides generate_spectral_shape because this method is not used in the VaryingSpectrum version of Model

VaryingSpectrum.make_spec(expr)¶

overrides make_spec because this method is not used in the VaryingSpectrum version of Model

VaryingSpectrum.spectral_shape_from_file(i)¶

support function for generate_light_curve_from_flux_and_spectrum, which takes files at user-specified time intervals to define a source’s spectrum

VaryingSpectrum.generate_light_curve_from_flux_and_spectrum()¶

calls spectral_shape_from_file(self,i) for spectral shape and generates the final light curve

SourceMap¶

The Simulator software comes with the following SourceMap class:

class SourceMap.SourceMap¶

a data structure for storing multiple sources from a single simulation.

The SourceMap class contains the following methods:

SourceMap.store_source(source)¶

stores a single source within the SourceMap

SourceMap.return_source_map()¶

returns the entire SourceMap

SourceMap.print_star_coords()¶

prints a regions list from the Sources stored in the SourceMap

SourceGenerator¶

The Simulator software comes with the following SourceGenerator class:

class SourceGenerator.SourceGenerator(rate=34, start=10000, end=22627000, distribution='no', print_locations='no')¶

provides a framework to simulate multiple source’s in the same run

The SourceGenerator object can call the following methods:

SourceGenerator.generate_sources()¶

generates a specific number of sources based on the length of the simulation and the user-defined source rate

SourceGenerator.one_source()¶

runs the simulations for exactly one source.

SourceGenerator.find_emission_start()¶

randomly selects a start time for a transient’s flare

SourceGenerator.find_galactic_longitude()¶

randomly selects a galactic longitude for a source

SourceGenerator.find_galactic_lattitude()¶

randomly selects a galactic latitude for a source

SourceGenerator.print_locations()¶

prints the text for a region file to console.

Plotter¶

The Simulator software comes with the following Plotter class:

class Plotter.Plotter(source, array, ntstep, nestep, tstart, ebin)¶

creates plots for the light curves of simulated sources.

The Plotter class contains the following methods:

Plotter.validation_report()¶

provides a series of graphs to review a source’s light curve. Prints to PDF file.

Plotter.more_detailed_validation_report()¶

provides an extremely detailed series of graphs to review a source’s light curve. Prints to PDF File

Controller¶

The controller class lets users call the other modules.
